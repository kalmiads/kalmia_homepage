# Experience 페이지 설계

날짜: 2026-07-15

## 목적
kalmia 랜딩 페이지의 메뉴에서 `Experience` 클릭 시 이동할 별도 페이지를 만든다.
디자인 작업/프로젝트를 카드 그리드로 보여주는 포트폴리오 페이지.

## 결정 사항
- **구조**: 별도 파일 `experience.html`. 기존 `css/style.css`, `js/main.js` 공유.
- **콘텐츠**: 지금은 샘플(더미) 카드 6개. 나중에 실제 작업으로 교체.
- **톤**: 랜딩과 동일한 어두운 배경(`#0d0a16`), 흰 글씨, 동일 폰트(Poppins/Aclonica).
  영상 배경 없음. 페이지 스크롤 허용.

## 레이아웃
1. **헤더**: 기존과 동일 — 좌측 로고(→ `index.html` 홈 링크), 우측 원형 메뉴 버튼.
   같은 어두운 메뉴 오버레이 재사용. 티커는 미포함.
2. **타이틀 영역**: "Experience" 제목(Poppins) + 한 줄 소개.
3. **작업 그리드**: 반응형(데스크톱 3열 / 태블릿 2열 / 모바일 1열).
   카드: 그라데이션 이미지 플레이스홀더 + 제목 + 카테고리. hover 시 살짝 떠오르고 이미지 확대.

## 재사용/연동
- `js/main.js`는 `.cursor`/`.menu-btn`/`#bg`에 가드가 걸려 있어 그대로 재사용 가능.
  단, 커서 블록은 `.cursor` 마크업이 필요하므로 새 페이지에도 커서 마크업 포함.
- 영상(`#bg`)은 이 페이지에 없음 → 해당 블록은 가드로 무시됨.
- 랜딩의 `Experience` 링크: `href="#experience"` → `href="experience.html"`.
- 새 페이지 전용 스타일은 `css/style.css` 하단에 `.exp-*` 네임스페이스로 추가.

## 향후 교체 포인트
- 카드의 플레이스홀더 블록 → 실제 이미지.
- 더미 제목/카테고리 → 실제 작업 정보.
